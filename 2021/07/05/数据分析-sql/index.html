数据分析--sql

发表于2021-07-05|更新于2021-07-09|数据分析
|字数总计:1.3k|阅读时长:4分钟|阅读量:

本篇内容记录 sqlzoo 中的一些笔记以及对于 sql 知识的整理。

基础知识

表的创建

  • 首先判断表是否存在，如果不存在，则创建
  • 主键设置，默认值设置

创建表的同时导入别的表中数据

批量插入数据，不使用多条INSERT语句

添加新字段

NULL值判断

与字段包含0、空字符串或仅仅包含空格不同。

  • column is NULL
  • column is not NULL

in与exists的区别

  • IN
    • 先查询内查询
    • 然后将返回结果和外表做笛卡尔积
    • 然后根据条件进行筛选
    • 一次检索
  • EXISTS
    • 指定一个子查询，检查行的存在
    • 遍历循环外表，然后看外表中的记录有没有和内表的数据一样的
    • 多次检索
    • 先检索外表–外部查询
    • 根据返回的结果，再去执行EXISTS内部语句
      • 如果成立，则存入结果集，直到遍历完成外表
      • 然后返回结果集

常用函数

CONCAT(data1, data2, data3)

  • 可以将data1 data2 data3拼接起来
  • 其中可以包含通配符，如果不想用通配符，可以加上转义字符\

REPLACE(str1, str2, str3)

  • 在str1中如果出现了str2,就使用str3代替

MID(column_name, start, stop)

  • column_name:要提取的字符的字段
  • start: 开始位置
  • stop: 要返回的字符数，可以省略，返回剩余文本

ROUND(digit, num)

  • digit 为需要截取的浮点数
  • num为保留的小数位数
  • 如果要找最近的1000
    • 可以先除以1000， 然后去掉小数位，接着乘以1000

ALL()

  • 逻辑运算符
  • 将单个值与子查询返回的单列值进行比较

CASE WHEN

  • \ 转义字符

case 列名

when 条件值1 then 选项1

when 条件值2 then 选项2

……

else 默认值 end

**case **

when 条件值1 then 选项1

when 条件值2 then 选项2

……

else 默认值 end as 列名

coalesce(exp1, exp2, exp3…)

  • 返回第一个非空值
  • 通常用于值的替换

group_concat(column)

  • 使用逗号将字段进行拼接
  • 通常配合group by使用

查询

**聚合函数(AVG MAX MIN SUM)**无法出现在WHERE子句中

  • 聚合函数是基于列数据进行计算的
  • where子句是对数据的行进行过滤
  • 从执行顺序也可以看出一二

having经常需要和聚合函数搭配使用

标准查询

  • LIMIT 1 OFFSET 1
    • 选择一个数据
    • 跳过第一行从第二行开始

模糊查询

  • %
    • 代表任意字符出现任意次数
  • _
    • 代表单个字符
  • 包含三个或者三个以上a
    • %a%a%a%
  • t作为第二个字母
    • _t%
  • 有两个字母o被另外两个字母相隔
    • %o_ _o%

子查询

在每一個州中找出最大面積的國家，列出洲份continent, 國家名字 name **及面積 **area。 (有些國家的記錄中，AREA是NULL，沒有填入資料的。)

  • 当需要重复使用一张表的时候需要重命名
  • 某些SQL系统，必须对子查询进行重命名

分组与汇总

HAVING与WHERE的区别

  • WHERE在数据分组前进行过滤，即在分组之前过滤数据，条件中不能包含聚合函数
  • HAVING在数据分组之后进行过滤，必须使用聚合函数
  • WHERE排除的行不包含在分组中
  • 使用MAX MIN可以代替ORDER BY

多个字段进行分组

  • GROUP BY X：将所有具有相同X字段的记录放到一个分组里面
  • GROUP BY X Y：将所有相同X Y字段的记录放在一个分组里面

表联结

left join(左联接)

  • 返回左表中所有记录和右表中联结字段相等的记录

right join(右联结)

  • 返回右表中所有记录和左表中联结字段相等的记录

inner join(等值联结)

  • ☞返回两个表中联结字段相等的记录

full join(外联结)

  • 返回两个表中的行：left join + right join

cross join

  • 笛卡尔积
  • 返回的行数=左表的行数*右表的行数

数据库在通过连接两张或者多张表来返回记录时，都会生成一张临时表，然后将这张临时表返回给用户。在使用left join时，on和where条件的区别如下：

  • on条件是在生产临时表时使用的条件，它不管on中的条件是否为真，都会返回左表中的记录
  • where是在临时表生成之后，再对临时表进行过滤的条件。条件不为真会全部被过滤掉
