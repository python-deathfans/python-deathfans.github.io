sql(总结)

发表于2021-07-10|更新于2021-09-07|数据分析
|字数总计:2.2k|阅读时长:8分钟|阅读量:

本篇内容对于之前的数据分析sql篇进行总结。

基础知识

sql执行顺序

  • from
  • join
  • where
  • group by
  • having
  • select(窗口函数在此处执行)
  • order by
  • limit

表的创建

  • 首先需要判断一下表是否存在，如果不存在在，则创建
  • 主键设置， 默认值设置

插入数据(判断是否存在)

创建表的同时导入查询数据

添加新字段

表的更新

  • set更新多个字段的时候使用逗号分隔

表的重命名

in和exists的区别

  • IN
    • 先执行子查询
    • 然后将结果和外表做笛卡尔积
    • 然后根据条件进行筛选
    • 一次检索
  • EXISTS
    • 先执行主查询的表
    • 遍历循环外部查询结果，一行一行待入子查询进行检索，如果返回结果，则将该行放入待返回的结果集

LIMIT

  • limit 2
    • 限制结果为前两条数据
    • 等价于 limit 0, 2
  • limit 2, 3
    • 从第2行开始，查询3条记录， 起始值为0行
  • limit 2 offset 3
    • 查询2条记录，从第3行开始，起始值为0行

行列转换

  • group by + case when

常用函数

CONCAT(data1, data2, data3)

  • 可以将data1..data3拼接起来
  • 其中可以包含通配符，如果不想用通配符，可以加上转义字符\
  • 如果任意一个参数为null，则结果返回null

REPLACE(str1, str2, str3)

  • 如果在str1中出现了str2,则使用str3进行替换

MID(column_name, start, stop)

  • column_name: 要提取字符的字段
  • start: 开始位置
  • stop: 要返回的字符数，可以省略，返回剩余文本

ALL()

  • 逻辑运算符
  • 将单个值和子查询返回的单列值进行比较

CASE WHEN

case 字段

WHEN 条件1 then 值1

WHEN 条件2 then 值2

ELSE 默认值

END 字段重命名

group_concat(column)

  • 使用逗号将该字段进行拼接
  • 通常配合group by使用

时间函数

  • cur_date()
    • 返回当前日期
    • 2021-7-18
  • weekofyear()
    • 返回日期所在的周
  • year()
    • 返回日期所在的年
  • date_add(date, interval)
    • 日期相加
    • interval可以是1 day 1hour 1 minite
  • date_sub(date, interval)
    • 日期相减
    • 用法类似于date_add
  • datediff(date1, date2)
    • 返回 date1-date2
    • 只有date的日期部分参与计算，时间部分不参与计算
  • date_format(date, format)
    • 时间格式转换
    • 将date转换为format格式的时间
  • weekday()
    • 返回每周的第几天

left, right substring函数

left(arg, 2)

  • 返回arg最左边的2个字符

right(arg, 2)

  • 返回arg最右边2个字符

substring(s, n, len)

  • 返回字符串s从第n个位置开始长度为len的字符串，没有len值，默认到最后一个字符

cast() 类型转换函数

cast(x as type)

  • 将字段x转为type类型变量

聚合函数

聚合函数自动忽略NULL行，所以手动计算平均值的时候需要注意

查询

标准查询

  • limit 1 offset 1
    • 选择一个数据
    • 跳过第一行从第二行开始选取

模糊查询

  • %
    • 代表任意字符出现任意次数
  • _
    • 代表单个字符
  • 包含三个或者三个以上a
    • %a%a%a%

子查询

在每一個州中找出最大面積的國家，列出洲份continent, 國家名字 name **及面積 **area。 (有些國家的記錄中，AREA是NULL，沒有填入資料的。)

  • 某些SQL系统需要对子查询重命名

分组与汇总

分组先当于对相同的字段进行合并汇总

多个字段进行分组

  • GROUP BY X: 将所有具有相同X字段的记录放到一个分组里面
  • GROUP BY X, Y: 将所有相同 X Y字段的记录放在一个分组里面

表联结

left join(左联接)

  • 返回左表中所有记录以及右表中联结字段相等的记录

inner join(等值联结)

  • ☞返回两个表中联结字段相等的记录

full join(外联结)

  • 返回两个表的并集

cross join

  • 交叉联结
  • 笛卡尔积

数据库在通过连接两张或者多张表来返回记录时，都会生成一张临时表，然后将这张临时表返回给用户。在使用left join时，on和where条件的区别如下：

  • on条件是在生产临时表时使用的条件，它不管on中的条件是否为真，都会返回左表中的记录
  • where是在临时表生成之后，再对临时表进行过滤的条件。条件不为真会全部被过滤掉

窗口函数

有什么用

  • 组内排名

partition by 与 group by的区别

  • group by 每个类别只有一行，改变了表的行数
  • partition by 每个类别返回对应的行数，不改变表的行数

写法

函数() over(partition by 列名1 order by 列名2 rows between n/current row)

  • rows between 限制窗口大小，不指定的话默认从起点到当前行
    • 1 preceding: 往前1行
    • 1 following: 往后1行
    • current rows: 当前行
    • unbounded preceding 分区第一行
    • unbounded following 分区最后一行

基本语法

  • <窗口函数> over(partition by <用于组内分组的列名> order by <用于排序的列名>)

    • 其中partition by 子句可以省略

  • 常见窗口函数

    • 专用(排序)
      • rank
        • 如果有并列占用下一名次
        • 1, 1, 1, 4
      • dense_rank
        • 如果有并列，不占用下一名次
        • 1, 1, 1, 2
      • row_number
        • 不考虑并列名次问题
    • 聚合函数(操作数据，截至到本行)
      • sum
      • avg
      • count
      • max
      • min

  • 注意点

    • 因为窗口函数是对where或者group by子句处理的结果进行操作，所以窗口函数原则上只能写在select子句中

聚合函数

  • sum
  • min
  • max
  • avg

排序函数

  • row_number
    • 不考虑并列问题
  • rank
    • 并列不连续
    • 1 1 3
  • dense_rank
    • 并列连续
    • 1 1 2

极值函数

  • first_value()
    • 取出当前分区下的第一个值
  • last_value()
    • 取出当前分区下的当前行的值，如果不指定range范围

移动函数

  • lag(a, 1, 0) over(partition by b order by c)
    • 针对的是a 这个字段
    • 1 代表 后移1行
    • 0代表第一个数据默认值是0,不写即为NULL
  • lead(a, 1, 0) over(partition by b order by c)
    • 针对的是a这个字段
    • 1 代表 前移一行
    • 0 代表最后一个数据默认值是0

中位数特征

  • 当个数为偶数的时候，起始位置为个数/2，结束位置为个数/2+1
  • 当个数为奇数的时候，起始位置为ceiing(个数/2)，结束位置和起始位置一样

mysql向上取整函数

  • ceil()
  • floor()
