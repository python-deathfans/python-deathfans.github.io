ssh公钥私钥使用问题

本篇文章的契机主要是最近配置博客遇到的两个问题：

  • 一台机器如何给两个github仓库配置公钥
  • .ssh路径下config文件的使用

公钥私钥基础知识

  • 公钥私钥创建命令

    • ssh-keygen -t rsa -f ~/.ssh/id_rsa.work -C "Key for Work"

ssh-keygen -t rsa -f ~/.ssh/id_rsa.github -C "Key for GitHub"

  • config文件用处

    • 用于指示不同的主机使用的私钥文件

    • 这样就可以解决一台电脑给多个github账号配置公钥目的

    • 文件格式

      • 将对应的公钥信息配置到github不同的账号，即可完成免密通信

      • 验证

        • ssh -Tv git@github-me
          • 查看是否访问成功
          • 不能使用github.com

      • config文件匹配过程

        SSH默认私钥文件(从上至下匹配)

        ~/.ssh/id_rsa
        ~/.ssh/id_dsa
        ~/.ssh/id_ecdsa
        ~/.ssh/id_ed25519
        ~/.ssh/id_ecdsa_sk
        ~/.ssh/id_ed25519_sk

        • 解析配置顺序：
          • 从文件顶部到底部逐行解析。
          • 每个 Host 块定义一组配置参数（如 IdentityFile, User, Port 等）。
          • 支持通配符（如 * 匹配任意字符，? 匹配单个字符）。
        • 匹配目标主机名：
          • SSH 会将命令行中的目标地址（如 git@github.com）与 Host 配置逐一比较。
          • 第一个匹配的 Host 块生效，后续匹配的 Host 块会被忽略。
        • 应用配置参数：
          • 使用匹配 Host 块中的 IdentityFile 指定私钥。
          • 若未指定 IdentityFile，则回退到默认私钥顺序。

hexo搭建博客注意事项

  • 如果购买了域名，在github仓库设置->Page设置中映射了域名，当域名失效的时候一定记得要删除映射，否则正常访问python-deathfans.github.io会失败(排查了半天，以为是阿里云购买域名时做的映射没有失效，一问才知道，当域名被回收之后，相关的所有配置也失效)
  • 如果完成第一步，发现还是进行了跳转，可以看一下hexo博客目录 source底下是否有一个CNAME文件，删除文件，清除浏览器缓存，再次浏览即可
