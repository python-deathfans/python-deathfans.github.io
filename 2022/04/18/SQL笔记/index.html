执行顺序

  • from
    • 得到单张表或者多张表的笛卡尔积(虚拟表)
  • on
    • 条件过滤
    • 剔除的是表连接之后的数据
    • 然后执行left join or right join，添加相应的记录
  • where过滤
    • 由于此时还没有进行数据分组，因此无法使用聚合函数进行过滤
    • 表别名也无法使用
  • group by 分组操作
  • having过滤
  • select操作
  • distinct子句
  • order by
  • limit

常见指令

  • 创建数据表
    • ```sql
      CREATE TABLE IF NOT EXISTS nowcoder_tbl(
      nowcoder_id INT UNSIGNED AUTO_INCREMENT,
      nowcoder_title VARCHAR(100) NOT NULL,
      nowcoder_author VARCHAR(40) NOT NULL,
      submission_date DATE,
      PRIMARY KEY ( nowcoder_id )
      )ENGINE=InnoDB DEFAULT CHARSET=utf8;
    • 更新多个字段时，使用逗号进行分隔
  • 去重操作
    • group by
    • distinct
  • 从另一个表中导入 数据
    • insert into tb_name select * from tb_name2 where key = value

常见函数

字符串函数

  • length()
  • right()
  • replace(string, old, new)
    • 将string中的old字符替换为new字符，并将替换之后的字符串返回
  • substring_index(str,delim,count)
    • str代表检索的字符串
    • delim代表分隔符
    • count代表第几次出现的位置：
      • if count > 0,则返回从左到右的字符串
      • if count < 0,则返回从右到左的字符串
  • substring(str, pos, len)
    • str代表检索的字符串
    • pos代表要截取的位置：
      • if pos>0, 则从左往右截取
      • 否则，从右往左截取
    • len代表截取长度，如果不传，则截取右侧剩余或者左侧剩余

日期函数

  • timestampdiff(interval, time_start, time_end)
    • 计算time_start - time_end之间的时间差
    • 单位以interval为准

if函数

  • ifnull(column, 3)
    • 如果column字段的值为null，则初始化为3
    • 否则不操作
  • if (sex=’m’, ‘f’, ‘m’)
    • 当sex=’m’时，返回’f’
    • 否则返回’m’

琐碎知识点

in和exists的区别

  • in代表外表和内表进行hash连接，且先执行子查询
  • exists先对外表进行循环，然后在内表进行查询

replace into与insert into的区别

  • replace into
    • 尝试插入数据
    • 如果存在该行数据，则先删除此行数据；否则直接插入新数据

truncate delete drop的区别

  • delete、truncate都是删除表数据，不删除表结构，drop直接删除表数据及结构
  • delete是DML语句，支持事务回滚；truncate和drop是DDL语句，操作完立马生效
  • 执行速度：drop>truncate>delete
