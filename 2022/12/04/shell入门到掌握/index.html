shell入门到掌握

shell中各种括号的使用

  • {}

    • 用于执行命令
    • 里面可以放置一串命令，彼此使用分号隔开
    • 第一个命令和左括号之间要有空格
    • 最后一个命令也要加上分号
    • var=”zhangsan”; { echo var;var=”lisi”;echo var;var=”lisi”;echovar;var=”lisi”;echo var;var=”lisi”;echovar;}; echo $var
    • 集合&展开
      • echo ab{a..c}
      • mkdir cd{c..d}
      • echo ab{a,b,c}
      • cp test.sh{,-bak}
    • 合并多个文件
      • cat {file1, file2, file3} > all.txt
      • 纵向合并
      • 横向合并
        • paste file1 file2 > all.txt
    • 输入输出重定向
      • {
        • } < file1
      • {
        • } > file2
      • 文件描述符
        • 0 标准输入
        • 1 标准输出
        • 2 标准错误输出
        • &>
          • 标准输出以及标准错误重定向

  • ()

    • 命令集合
    • 命令替换 == 反引号
      • $(ls -l) == ls -l
    • 初始化数组
      • (a b c d)
    • 命令组
      • (a=hello**;** echo $a)
      • 会开启一个子shell，因此父shell中无法访问子shell的内容

  • (())

    • 专门用来进行整数运算的命令，不能对小数或者字符串进行运算

    • 一个或者多个都可以，中间使用,进行分隔

    • 还可以进行比较运算，自增运算

    • 总之，涉及到整数数值运算的，优先使用(())

  • []

    • 主要用于条件判断

    • 两个符号左右都需要有空格

    • 操作变量左右也要有空格

  • [[]]

    • 主要用于条件判断

    • 符号左右要有空格

    • 操作变量左右也要有空格

    • 更加通用

基础命令

  • &
    • 后台执行
    • jobs
      • 查看正在后台执行的命令
  • echo
    • -n不换行
    • -e 解析转义字符
  • export
    • 把作为它参数的变量导出到子shell中，并使之在子shell中有效
  • shift
    • 把所有参数左移一个位置，$0不变，直到$1也没有
  • test命令 等价于 []的判断
    • 文件
    • 字符串
    • 数字
  • 多条命令执行
    • 不同命令之间;分隔

SHELL

Shell终端解释器当作人与计算机硬件之间的“翻译官”，它作为用户与Linux系统内部的通信媒介，除了能够支持各种变量与参数外，还提供了诸如循环、分支等高级编程语言才有的控制结构特性。

最简单的脚本

  • 需要声明解释器路径
    • #! /bin/bash
      • 不确定的情况下，可以执行 echo $SHELL
    • 输入一些简单的命令
      • pwd
      • ls -al
      • ps -aux
  • 执行方式
    • 绝对地址和相对地址
      • 需要脚本有可执行权限
      • 使用的是什么解释器 #! /bin/bash
      • chmod +x test.sh
    • bash sh test.sh
      • 不需要有可执行权限
      • sh 则使用 /bin/sh解释器
      • 脚本中可以不去声明 解释器
    • . test.sh
      • 不需要有可执行权限
      • 解释器权限继承自 bash
    • source test.sh
      • 常用于配置文件的生效
      • 不需要脚本有可执行权限

接收用户的参数

  • $0
    • 脚本名称
  • $#
    • 代表位置参数的个数
  • $*
    • 所有位置参数的值
  • $数字
    • 代表对应位置的参数值
  • $？
    • 代表上一个执行结果的返回值

判断用户参数

[ 表达式 ]

  • 两端均有空格

  • 0 代表条件成立

  • 非0 代表条件不成立

  • 文件测试语句

    • -d
    • -e
    • -f
    • -r
    • -w
    • -x

  • 逻辑测试语句

    • 与

      • &&

      • 表示当前的命令执行成功后才会执行后面的命令

    • 或

      • ||
      • 表示前面的命令执行失败才会执行后面的命令

    • 非

      • !

  • 数值测试语句

    • -eq
    • -ne
    • -gt
    • -lt
    • -le
    • -ge

  • 字符串测试语句

    字符串判断的时候一定要加上”$str”，否则会出现一些奇怪的错误

    • =
    • !=
    • -z
      • 判断字符串是否为空

数组

  • 动态数组
    • array=($(command))
    • 动态生成
  • array[*]与array[@]的区别
    • *传入的数组会被当作一个整体，@会识别所有的元素

开发规范

  • #! /bin/bash
    • 声明解释器
  • 括号成对出现
  • 代码缩进；整块代码保持统一缩进
  • 字符串赋值给变量要加双引号，并且等号前后不能有空格
  • 变量命名
    • 全局变量
      • 变量名全部大写;多个单词使用_连接
      • 最好使用export进行定义
    • 局部变量
      • 可以使用驼峰命名：首单词字母小写，其余单词首字母大写
      • 函数中的局部变量可以使用local去修饰，使其在本函数内部有效
  • 变量引用规范
    • 若引用变量的前后都有英文字符，则需要使用${param}的方式
  • 函数命名及定义规范
    • 大驼峰命名
      • CreateFile()

调试

  • sh -[nvx] file.sh
    • -n
      • 只检查语法，不执行
    • -x
      • 将执行脚本的内容及输出显示到终端，是一个非常好用的参数

函数

  • 定义
    • function func(){}
    • function func{}
    • func(){}
  • 调用
    • 无参
      • func
    • 有参
      • func $parm1 $parm2
    • 如果想要将函数返回值赋值给另一变量，可使用$(func)
  • 返回值
    • shell中的返回值是命令的执行状态，返回值为0，代表执行成功，返回值为非0代表执行出错
    • 要想获得真实的返回值，在函数中 echo $return_value

环境变量

  • 环境变量

    • 全局

      • 父子shell都是可见的

      • env printenv来查看所有的环境变量

      • 全局环境变量在设置该变量的父进程所创建的子进程中都是可见的。创建全局环境变量的方法是先创建局部变量，然后再将其导出到全局环境中。

      • export param

      • unset 删除环境变量

    • 局部

      • 只能在定义它的进程中可见

      • set命令可以查看所有的变量，包括全局和局部，因此env和printenv中可以看到的变量使用set依然可以看到

  • PATH环境变量

    • 当你在shell中输入一个外部命令时，shell必须搜索系统，从中找到对应的程序。PATH环境变量定义了用于查找命令和程序的目录。

    • echo $PATH

    • PATH中的目录之间以冒号分隔

    • 如果希望程序位置也可用于子shell，则务必确保将修改后的PATH环境变量导出

      • export PATH=$PATH

文件导入

为了提高代码复用率，可以在脚本中导入其他文件

两种方法

  • . file
  • source file
