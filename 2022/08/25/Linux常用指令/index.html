文本处理三剑客

grep

linux中强大的文本搜索指令，可以配合正则表达式，达到高效的搜索

  • -l

    • 以指定字符串查找对应的文件名

  • -i

    • 忽略字符大小写

  • -v

    • 反向查找
    • 查找不匹配的行

  • -n

    • 打印行号

  • -r

    • 递归查找

  • -A100

    • After

  • -B100

    • Before

  • -C100

    • 两者兼得

  • -q

    • quiet
    • 不输出任何内容

  • 逻辑操作

    • 与

      • .*

    • 或

      • -E |

    • 非

      • -v

  • -w

    • 精确匹配
    • -w 234
      • 只匹配234

  • -o

    • 只输出匹配内容

  • -c

    • 输出匹配到的行数，而不是匹配到的次数

  • 正则

    • [] 搜索集合

      • find . -name “*.txt” -type f |xargs grep -n “he[il]”

    • ^符号做[]内的前缀，表示除[]内的字符之外的字符

    • [] 内可以用范围表示，比如[a-z] 表示小写字母,[0-9] 表示0~9的数字, [A-Z] 则是大写字母们。[a-zA-Z0-9]表示所有数字与英文字符

    • 行首与行尾字符 ^ $. ^ 表示行的开头，$表示行的结尾( 不是字符，是位置）那么‘^$’ 就表示空行,因为只有行首和行尾。

    • grep -n ‘^[a-z]’ regular_express.txt

      • 搜索以小写字母开头的内容

    • grep -n ‘^[^a-zA-Z]’ regular_express.txt

      • 搜索非英文字母开头的内容

    • 任意一个字符. 与重复字符 *

      • *代表>=0个

      • .表示任意一个

      • .* 表示任意多个字符

      • +前面的字符至少出现一次

    • []{}

      • [0-9]{2}
      • 0-9数字任取两次

    • [0-Z]

      • 表示数字和字母

  • 查找30分钟内被修改过的so文件和bnd文件

    • find ~ -type f -mmin -30 | grep -E “so$|bnd$”

sed

sed 会根据脚本命令来处理文本文件中的数据，这些命令要么从命令行中输入，要么存储在一个文本文件中，此命令执行数据的顺序如下：

  1. 每次仅读取一行内容；
  2. 根据提供的规则命令匹配并修改数据。注意，sed 默认不会直接修改源文件数据，而是会将数据复制到缓冲区中，修改也仅限于缓冲区中的数据；
  3. 将执行结果输出。

当一行数据匹配完成后，它会继续读取下一行数据，并重复这个过程，直到将文件中所有数据处理完毕。

  • 常用选项

    • -n
      • 和命令符p结合使用
      • 用于打印
    • -i
      • 直接修改文件内容，而不是输出到终端

  • 追加操作

  • 替换命令

    • [address]s/pattern/replacement/flags

      其中，address 表示指定要操作的具体行，pattern 指的是需要替换的内容，replacement 指的是要替换的新内容。

    • sed ‘s/test/trial/2’ data4.txt

      • 只替换第二次匹配到的

    • sed ‘s/test/trial/g’ data4.txt

      • 全部替换，每行中所有匹配到的

  • 删除命令

    • [address]d
    • sed ‘3d’ data6.txt
      • 删除第3行
    • sed ‘2,3d’ data6.txt
      • 删除2，3行
    • sed ‘3,$d’ data6.txt
      • 删除第3行到结尾之间的内容

  • 打印

    • sed -n ‘100,200p’ file

  • 寻址方式

    1. 以数字形式指定行区间；
    2. 用文本模式指定具体行区间。
    • sed ‘2s/dog/cat/‘ data1.txt
      • 只替换第二行的内容
    • sed ‘2,$s/dog/cat/‘ data1.txt
      • 修改第2行到结尾行的内容
    • 文本寻址
      • 从包含world的行开始，到包含love的行结束，将其中的you替换为yapp。
      • 针对所有包含very的行，将其中的very替换为空格，-i代表修改原始文件，而非缓冲区数据
      • /System/d
        • 删除包含System的行
      • /System|Linux/d
        • 删除包含System或者Linux的行
      • /^i/d
        • 删除以i开头的行
      • /^ /d
        • 删除以空格开头的行
      • -n ‘/love/p’
        • 打印包含love的行
    • sed -e
      • 执行多条命令
      • ‘s/red/black/g; s/cat/dog/g’

find

linux中一切皆文件，如何查找文件是学习Linux必须掌握的技能

  • -maxdepth

    • 1
    • 子目录的深度
    • 在路径后面，如果在其他位置，则会报错

  • find 路径 -name ‘文件名’

    • find /home -name “*.c”
    • 查找/home目录下的所有以.c为后缀的文件

  • -type

    • d
      • 文件夹
    • f
      • 普通文件

  • -size

    • +10M
      • 超过10M的文件

  • atime

    • 读取时间

  • ctime

    • change time
    • 文件权限修改时间

  • mtime

    • modify time

    • 文件修改时间

    • 单位是天

  • mmin

    • 单位是分钟

  • -user

    • 用户

  • -group

    • 属组

  • -perm

    • 按照文件权限查找
    • 755

  • !

    • 取反
    • 比如，某个路径下除了一个文件其余全部删除，可以使用这种方法

  • -perm

    • 按照权限查找文件

    • -744

      • 表示至少744权限

    • find . -perm -744 -type f -name “*.sh”

  • -not

    • 取反
    • 补集
    • 用于排除一些文件

-exec

  • 类似与管道符
  • 将前面的输出输入到后面的命令
  • 必须加上 {} ;

-ok

  • 功能和-exec类似，只不过每个操作都需要用户去确认

新手必须掌握的Linux指令

常用系统工作命令

  • echo

    • 将字符串的值重新在终端中进行显示
    • 使用$param，可以将param变量的值打印在终端中
    • -e
      • 转义字符
    • -n
      • 不进行换行输出

  • reboot

    • 重启

  • poweroff

    • 关机

  • shutdown

    • -r
      • 重启
    • -h
      • 关机
    • +1
      • 表示1分钟之后重启或者关机
    • -c
      • 取消最近的一个关机或者重启任务

  • ps

    • process status
    • 进程状态
    • -a
      • 列出带有终端的所有用户的进程
    • -u
      • 面向用户友好的显示风格
    • -x
      • 列出当前用户的所有进程
    • -e
      • 列出所有进程
    • -f
      • 显示完整格式的进程列表

  • 终止进程

    • kill PID -9

      • 强制终止

    • killall 进程名称

      • 范围终止
      • 带有进程名称的进程全部杀死

  • top

    • 动态监视系统运行状态及负载状态
    • 类似于windows 任务管理器

  • pidof

    • 查询某个服务的进程数值
    • 常用于杀死对应的进程

  • date

    • 显示系统时间和日期
    • 修改日期和时间
      • root用户
      • date -s ‘20230217’
        • 只修改日期，同时时间重置为00:00:00
      • date -s ‘18:09:34’
        • 只修改时间
    • 格式化显示
      • date +%F
        • 2023-02-19
      • date +”%Y-%m-%d”
      • date +”%Y%m%d”

  • crontab 定时任务

    • 格式
      • 分 时 天 月 星期 command
    • 服务启动
      • service crond start
      • systemctl start crond
    • 定时任务没有启动排除步骤
      • 首先，手动执行脚本，看看有没有语法错误；
      • 其二，查看定时任务服务是否启动；
      • 其三，所有的路径一定要加上绝对路径；
      • 其四，环境变量问题，需要引入一些配置文件，否则某些命令无法识别；
      • 其五，结合日志查看错误信息，日志目录在 /var/log/cron
      • 六，代码中遇到每条执行的命令最好判断一下执行状态，出错的时候需要进行一下异常处理

  • seq 14

    • 序列，产生1-14 这十四个数

  • rename

    • 批量重命名
    • from to file
      • from原始文件名称
      • to 修改后文件名称
      • file 要修改的文件
    • ubuntu
      • from to 合到一起
      • 类似于sed替换
      • rename ‘s/old/OLD’ *.jpg

  • md5sum

    • 为文件创建md5码
    • md5sum hello > hello.md5
    • md5sum -c hello
      • 检查文件是否修改过

系统监测命令

  • free -h

    • 查看系统内存使用情况

  • who

    • 查看当前登录终端的用户信息

  • history

    • 查看之前敲过的命令
    • 以日志的形式存在的
    • -c
      • 参数表示清空日志

  • watch

    • 监视命令执行情况
    • 可以以全屏的方式动态显示命令的执行情况
    • -n
      • 命令的执行时间间隔，默认时2s
    • -d
      • 高亮显示命令结果的变动之处
    • watch -n 1 -d cat file.txt
      • 每隔一秒高亮显示file.txt文件的变动

  • alias

    • 别名；重命名
    • alias ls=’ls -l’
    • alias可以查看所有的重命名指令
    • 永久生效
      • 将alias ls=’ls -l’添加到~/.bashrc
      • 执行 source bashrc

  • du

    • 磁盘占用情况
    • disk usage
    • 主要用于查看目录或者文件大小
    • -h
      • 人性化阅读
    • -s
      • 只显示当前路径的总占用
      • du - sh
        • du -h –max-depth=0
        • 等价于 du -sh
    • –max-depth=1
    • –exclude
      • 去除某些目录
      • 绝对路径打包，就使用绝对路径排除
      • 相对路径打包，就使用相对路径排除
      • 如果要排除的是目录，不能以/结尾，否则排除无效
      • tar -czvf tmp.tar.gz –exclude shell/n*.txt shell
        • 注意位置
      • 匹配模式
        • 部分路径被匹配到 ，则整个路径被匹配到
        • 路径分量
      • 通配符
        • ?
          • 单个字符
        • *
          • 任意多个任意字符

  • df

    • disk free
    • 空余磁盘
    • -h
      • 人性化显示

  • 关机/重启

    • shutdown -h now

    • shutdown -r now

    • shudown -h +1

      • 1分钟之后关机

    • shutdown -h 11:00

      • 11：00关机

    • exit / logout

      • 退出当前用户
      • 或者 ctrl + d

  • pgrep

    • -u bzl

      • 查询bzl相关的进程

    • crond

      • 查询crond相关的进程

  • nohup &

    • 联合使用
    • 后台执行；退出登录依然会运行

  • jobs

    • 只能查看本终端开启的后台任务
    • -l
      • 显示PID进程号

  • service

    • 用于对系统服务进行管理，比如 start stop restart reload status等，该命令在centos7中被systemctl取代
    • service –status-all
      • 查看所有服务
      • 会显示哪些服务正在running

文件&目录操作

文件描述符

  • 0 标准输入
  • 1 标准输出
  • 2 标准错误输出
  • kill -l 1234 > /dev/null 2>&1
    • 该命令的输出到linux回收站，错误输出也是到该文件；&1代表第一个位置参数

  • pwd

  • cat

    • 打印文件内容

    • 多个文件合并

      • cat {file1, file2} > all.txt
      • cat file1 file2 > all.txt

    • -n

      • 显示行号

    • 清空文件

      • cat /dev/null > file

  • mkdir

    • -p
      • 递归创建

  • touch

    • 创建空文件

    • 修改文件时间

      • atime

        • 访问时间

      • ctime

        • 权限修改时间

      • mtime

        • 文件修改时间

    • touch file

      • 如果file存在，则修改了文件的mtime
      • 也就是ls -l 中显示的时间

  • cd

    • cd ~
      • 切换到家目录
    • cd -
      • 切换到上一次目录
    • cd ..
      • 切换到父级目录

  • ls

    • 显示当前目录下的文件信息
    • -A
      • 显示所有隐藏文件，不包括 . ..
    • -F
      • 目录会被加上/符号
    • -a
      • 显示全部文件，包括隐藏文件。Linux中隐藏文件以.开头
    • -l
      • 更全面的显示文件信息：文件大小、创建时间、所属用户、文件类型等
    • -d
      • 只显示当前目录的文件夹信息，不包括他们的子目录
    • ls -rtl
      • 按照修改时间升序排列文件，以列表形式显示
    • ls get
      • 显示包含get的文件名称
    • -R
      • 递归显示目录中子文件或者子目录
      • 不如 tree命令好用
    • 统计目录下所有的目录个数
      • ls -lR | grep ^d | wc -l
    • 统计目录下所有的文件个数
      • ls -lR | grep ^- | wc -l

  • cp

    • -r

      • 递归复制

    • -p

      • 保持文件原有信息
      • 尤其是mtime属性

    • -d

      • 如果复制的是符号链接，那么只复制链接本身

    • -a

      • 等价于 -pdr

    • -f

      • 强制复制，如果文件已经存在，和直接覆盖

    • -v

      • 显示复制详情

    • 骚操作

      • 当复制路径只有最后文件名不一致，可以采用
      • cp ~/bzl/file{, .bak}
      • 等价于 cp /bzl/file /bzl/file.bak

  • mv

    不会修改文件的修改时间，mtime

    • 移动或者重命名文件

      • -r

        • 递归移动

      • -i

        • 如果文件存在，则会提示是否覆盖

  • rm

    • -f
    • -r
    • -i
      • 删除之前需要用户进行确认

  • xargs

    由于很多命令不支持通过管道符进行参数传递，xargs的出现就是为了解决上述问题；

    默认适用的命令是 echo。

    • echo one two | xargs -n1 cp -v ./*.log

      • 将当前路径下的log文件拷贝到one two两个文件夹中

    • -n

      • 指定传入参数个数

    • -p

      • 交互，每个命令都会提示

    • -i

      • 使用{}替代之前的内容

    • -0

      • 配合find中-print0参数去使用
      • 解决文件名称含有空格的场景

    • 多行输出变一行

      • xargs < test.txt

  • 链接

    • ln 无参数 源文件 目标文件

      • 硬连接

      • 通过索引节点(inode)进行链接

        • ls -i
        • 可以查看到文件的索引节点

      • 多个文件名指向同一个inode

      • 只有删除了源文件以及所有的目标文件，文件实体才会被删除

    • ln -s 源文件 目标文件

      • 软连接
      • 有点像win中的快捷方式
      • 文件类型是l

  • dirname

  • basename

  • wc

    • -L

      • 打印宽度最大的行的长度

    • -l

      • 打印行数

    • -t

  • wc

  • iconv

  • cut

      • 遇到目录，最后一个目录后面不要加/

      • 比如，要排除当前路径下的a文件夹，–exclude=a

      • 比如，排除/root/a/c的文件夹，–exclude=/root/a/c，c后面不要加/

      • 打包的目录使用相对路径，排除的文件只能接相对路径

      • 打包的目录使用绝对路径，排除的文件接相对路径或绝对路径

      • 建议打包的目录和排除的文件使用的使用路径时，保持一致，都是用绝对路径，或都是用相对路径

      • tar -czvf tmp.tar.gz –exclude=*.log linux/

        • 打包的时候要回到上一级目录

        • 排除的时候由于是位置参数，所以需要写在路径的前面

    • tar tzvf tmp.tar.gz

      • 查看压缩包中的内容

      • 简洁命令查看

        • tar tf tmp.tar.gz

  • awk

    一行为一条记录，每个单词是一个字段

    • -F

      • 指定分隔符，默认是空格

    • ‘$0 !~/this/ {print}’

      • 打印不包含this的行

    • ‘$0 ~/^this/ {print}’

      • 打印不以thi开头的行

    • ‘$0 ~/this$/ {print}’

      • 打印以this结尾的行

    • ‘NF!=0{print}’

      • 打印非空行

    • ‘$0 != NULL {print}’

      • 打印非空行

    • if else判断

      • awk ‘{if ($0 == “hh”) hh+=1; else if ($0 == “hihi”) hihi+=1} END{printf “hh %d, hihi %d \n”, hh, hihi}’

    • FS

      • 字段分隔符，默认是空格
      • FS=””,将每个字符作为一个字段

    • RS

      • 记录分隔符
      • 默认是\n

    • NF

      • 当前记录中的字段数

    • NR

      • 行号

    • OFS

      • 输出字段分隔符
      • 这个字段的使用需要谨慎，可能并不会如你所愿

    • ORS

      • 输出记录分隔符

    • awk -F:

      • 等价于 awk ‘BEGIN{FS=”:”}’

    • 格式化输出

      • awk ‘{printf(“%.3f”, SUM/N)}’

    • 常用函数

      • index(str, substr)
        • 如果str包含substr，则返回对应下标(下标从1开始)，否则返回0
      • length()
        • 返回字符串长度
      • substr()
        • 字符串截取

    • 流程控制

      • if
        • if与else之间使用;进行分隔
      • for

        • 其中bandwidth是awk中的数组

    • 数组

      • for (i in arr) printf i “\t” arr[i] “\n”

文件权限

文件有对应的所有者、所有组以及其他人所拥有的权限，分为可读(r)、可写(w)以及可执行(x)权限；

对于目录而言

  • 可读：能够读取目录下的文件列表
  • 可写：能够在目录内新增、删除、重命名为文件
  • 可执行：能够进入目录

用户管理

  • useradd

    • 添加新用户

      • david是用户名
    • -M
      • 不创建家目录
    • -u
      • 指定用户的uid，不能重复
    • -g
      • 指定用户的属组，必须存在
    • 相关文件
      • 向/etc/passwd（用户文件）和/etc/group（组文件）文件内添加新用户和新用户组记录，向/etc/shadow（用户密码文件）和/etc/gshadow（组密码文件）文件里添加新用户和组对应的密码信息的相关记录

    • 用户信息修改

    • -d

      • 修改用户的家目录

    • -g

      • 修改用户的属组
      • 属组只有一个，但是可以有许多的附属组

    • -a -G

      • 将用户追加到其他附属组，而不离开其他组

    • -s

    • -u

      • 改变用户的uid

    • -l

      • 修改用户账号名称

    • -L

      • 锁定用户密码，不让修改

    • -U

      • 解除密码锁定

  • userdel

    • -f

    • -r

  • groupadd

    • -f

      • 强制覆盖，如果原组存在

    • -l

    • -u
      • 解除锁定
    • -d
      • 删除用户密码，使得密码为空
