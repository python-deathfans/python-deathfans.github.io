SQL必知必会

发表于2020-11-29|更新于2020-11-29|阅读数据分析
|字数总计:406|阅读时长:1分钟|阅读量:

本篇记录《SQL必知必会》这本书的学习以及重要的笔记。

  • 语句最好以分号结尾
  • 关键字大写，列名和表名小写
  • SQL不区分大小写

基础指令

  • 创建数据库
    • CREATE DATABASE 数据库名字;
    • CREATE DATABASE IF NOT EXISTS 数据库名字 DEFAULT CHARSET UTF8;
  • 选择数据库
    • USE 数据库名字；
  • 登录数据库
    • mysql -u root -p pwd
  • 删除表
    • DROP DATABASE 数据库名字；

检索数据

  • 检索唯一不重复的值

    • SELECT DISTINCT col FROM table;

  • 限制检索的长度

    • SELECT col FROM table LIMIT num;

  • 指定开始位置

    • SELECT col FROM table LIMIT num OFFSET num;
    • 从第num行开始检索num行数据，num从0开始。

排序检索数据

默认是升序，如果想要降序排列，使用DESC关键字进行限定。

  • SELECT prod_name FROM products ORDER BY prod_name;
    • ORDER BY关键字一定要放在语句的最后，不然会报错。
    • 通常情况下，ORDER BY使用的是显示的列，实际上并不一定要这样，用非检索列排序也是完全合法的。

过滤数据

  • 不匹配检查
    • !=
    • <>
    • 一般情况下两者可以互换，具体情况还是根据DBMS的文档决定
  • 范围值查询
    • BETWEEN .. AND ..
  • 空值检查
    • IS NULL

高级数据过滤

  • 多个子句结合
    • 使用逻辑运算符 AND OR NOT进行连接。当使用多个逻辑运算符时，最好加上括号，这样逻辑表达更加的清楚。
  • IN操作符号
    • 相当于完成了OR的功能
    • 优点
      • 操作速度比OR快
      • 逻辑更清楚
      • 操作顺序更容易管理
