git基本操作

发表于2020-12-22|更新于2020-12-22|随笔
|字数总计:807|阅读时长:2分钟|阅读量:

本篇记录一下git的基本操作。

全局配置命令

配置全局用户名、邮箱

  • git config –global user.name
  • git config –global user.email
  • git config –global user.password

git config –list

  • 查看配置列表

生成git 公钥

  • ssh-keygen -t rsa -C “xxx@xxx.com“
  • 拷贝.pub文件的公钥到github
  • 即可完成公钥的配置，免得每次push都需要输入用户名和密码

创建本地仓库

手动创建

  • git init
    • 初始化仓库
    • 生成.git文件夹

自动创建

  • 先创建远端仓库
  • git clone 远端仓库

本地仓库与远程仓库建立连接

  • 手动
    • git remote add origin 仓库地址
      • 本地仓库与远端仓库建立连接
      • 远端仓库的名字是origin
  • 自动
    • git clone 远端仓库
  • git remote -v
    • 查看远端仓库

提交到远程仓库完整步骤

  • 创建本地仓库
  • 本地仓库与远端仓库建立连接
  • git add a.js
    • 将工作区文件添加到暂存区
  • git commit -m ‘first commit’
    • 将暂存区文件提交到本地仓库
    • -m 显示的是提交信息
  • git push -u origin master
    • 将本地仓库所在分支的内容提交到远端仓库的master分支
    • -u 可以在推送的同时，将origin仓库的master分支设置为本地仓库当前分支的upstream(上游).添加了这个参数，将来运行git pull命令从远程仓库获取内容时，本地仓库的这个分支可以直接从origin的master分支获取内容，省去其他参数的麻烦。

分支的概念及应用

一个项目多人开发，我的任务需要开发一个功能，我没办法短时间内就开发测试等等一系列完成。所以我可以新建一个分支，然后把每次完成的代码都上传到这个分支里面去，最后等到所有操作都完成后，我再把这个分支整合到master中去

  • git branch
    • 查看当前分支
  • git branch -a
    • 查看本地仓库和远端仓库的所有分支
  • git checkout -b dev
    • 创建dev分支并切换到dev分支，相当于下面两个指令
      • git branch dev
      • git checkout dev
  • 这个时候所作的操作都是在分支dev下的，对于master分支没有任何影响，只是最后push的时候需要注意:git push origin dev。意思是提交到远端仓库的dev分支下。

分支合并

我们在dev分支下折腾完成，最终要整合到master分支：

  • git checkout master
    • 切换到master分支
  • git pull origin master
    • 将远端仓库的最新代码pull下来，因为期间其他人可能已经提交代码到远端master了，所以首先更新自己本地的master代码
  • git merge dev
    • 将dev分支下的操作合并到master分支下

注意事项

  • 为了让我们的代码是最新的状态和提交时不发生冲突，先执行
    • git pull origin master
      • 把远端仓库的master分支内容拉到本地仓库
      • 如果使用了git push -u origin master，则可以直接使用git pull命令将远端仓库的master分支内容直接拉到本地仓库.
