计算机二级-C语言

发表于2023-01-08|更新于2023-03-06|C语言
|字数总计:2.1k|阅读时长:6分钟|阅读量:

笔记

一、程序设计基本概念

1.1 程序和程序设计

  • 把高级语言编写的程序称为“源程序”，把由二进制代码表示的程序称为“目标程序”，通过编译程序把源程序转换成机器能接受的目标程序的过程叫做编译。
  • C源程序经过C编译程序编译之后生成一个后缀为**.obj的二进制文件，然后由称为“连接程序”（Link）的软件，把此.obj文件与C语言提供的各种库函数连接起来生成一个后缀为.exe**的可执行文件。
    • 编译
      • .c -> .obj
    • 链接
      • .obj -> .exe
      • 可执行文件

1.2 算法

算法是指为解决某个特定问题而采取的确定且有限的步骤

  • 特点
    • 有穷性
    • 确定性
    • 可执行性
    • 0个或者多个输入
    • 至少一个输出
  • 描述
    • 伪代码
    • 流程图

1.3 结构化程序设计和模块化程序设计

  • 结构化程序设计(C语言属于此种)
    • 顺序
    • 选择
    • 循环
  • 模块化程序设计
    • 计算机在处理复杂任务的时候，将复杂问题分解成若干个子任务，每个子任务又可以分成很多个小子任务，每个小子任务用一个个小模块来实现这些功能，每个程序员分别完成一个或多个小模块，这种程序设计方法叫做“模块化”的方法。
    • C程序的模块化主要通过函数来实现。C语言允许对函数单独进行编译，从而可以实现模块化

二、C语言设计的基础知识

  • C语言中，最基本的数据类型包括整型、实型、字符型

  • 整形常量

    • 十进制基本常量用一串连续的数字表示，如32767、-32768、0等；
    • 八进制也用一串连续的数字表示，其开头必须是数字“0”，如016等；
    • 十六进制数用数字0和字母X开头，其字母不区分大小写，如0Xf等。

  • 整数在内存中的存储形式

    • 正整数

      • 以原码的形式存储

    • 负整数

      • 以补码的形式存储

        计算方式

        • 原码按位取反

        • 加一

  • 赋值运算符的优先级只高于逗号运算符，比任何其他运算符的优先级都低，且具有自右向左的结合性

  • 运算符优先级

    • 非>算术运算符>关系运算符>与>或>赋值运算符

    • 条件运算符优先于赋值运算符，但是低于关系运算符

三、顺序结构

  • printf函数

    • “%8.6f”代表总共8位宽度，包括小数点，小数点后有6位小数

    • 数据都隐含右对齐，如果想左对齐，可以在格式控制中的“%”和宽度之间加一个“-”号来实现

    • 格式控制%m.nf，表示数据输出总的宽度为m位，其中小数部分占n位。当数据的小数位多于指定宽度n时，截去右边多余的小数，并对截去的第一位小数做四舍五入处理；而当数据的小数位少于指定宽度n时，在小数的右边补零；当m小于有效位数时，整数部分输出所有有效数字并且自动对齐，小数部分按照n指定位数输出

四、选择结构

  • switch语句以及case语句

    • default代表所有case标号之外的符号，可以出现在语句体中的任何位置

    • 若有与该值相等的标号，则执行标号后开始的各个语句，包括其后所有的case和default语句，直到结束；

    • 若没有与case相等的标号，但是存在default语句，则执行其后面所有的语句；

    • 如果没有与case相等的标号，也没有default语句，则不进入switch语句；

五、字符型数据

  • 字符常量只能用单引号括起来
  • C语言规定，所有的字符常量都作为整形来处理。字符常量中内存中只占一个字节，存放的是对应的ASCII代码值
  • 反斜线后的八进制可以不用0开头；十六进制只可以由小写字母x开头，不允许用大写字母X，也不能用0x
  • 字符串常量是由双引号括起来的一串字符。在C语言中，系统在每个字符串的最后自动加入一个字符\0作为字符串的结束标志

六、函数的调用

  • 若被调用的函数返回值为int或者char类型，被调用函数的定义可以放在调用位置之后

  • 用户定义函数必须遵循“先定义后使用”的原则，凡是在调用前未定义的函数，编译程序默认返回值为int类型

  • 函数调用间的数据传递

    • 实参和形参之间进行数据传递。数据只能从实参单向传递给形参，形参的改变不会改变对应实参的值

    • 被调用函数通过return语句把函数值返回给调用的函数

    • 通过全局变量共享数据

七、地址和指针

  • 计算机的内存是以字节为单位的一片连续的存储空间，每个字节都有一个编号，叫做内存地址。内存中的地址号是连续的，采用二进制表示
  • 指针的移动是指地址进行的增减运算，指针移动的最小单位是一个存储单元，而不是一个字节。所以，对于类型不同的指针变量，其地址值增减1所跨越的字节数是不同的
  • 32位机器，指针类型变量占4个字节，64位机器，指针变量占8个字节。sizeof 结构体类型变量，大小必须是最大变量的整数倍，不足需要进行补齐，这就是结构体内存对齐。

八、数组

  • 系统为定义的数组在内存中分配一串连续的存储单元时，这些存储单元中并没有确定的数值，可以采用边定义边赋值的方式

  • 第一个初值必定赋给标为0的元素，当所赋值少于定义数组的元素个数时，系统将自动给后面的元素补以初值0

  • 数组名可以认为是一个存放地址值的指针常量名，其地址值是数组第一个元素的地址，基类型为定义数组时的类型

  • 二维数组名作为实参和形参之间的数据传递当二维数组名作为实参时，对应的形参必须是一个行指针变量

    • 列下标不可缺少

    • 系统只为形参分配一个存放地址的存储单元，而不是分配存放所有数组元素的存储单元

进阶

编译 链接

  • gcc -c main.c
    • ./a.out
  • gcc -o main main.c
    • ./main
  • 声明和定义的区别
    • C语言中的声明（Declaration）有变量声明、函数声明和类型声明三种。如果一个变量或函数的声明要求编译器为它分配存储空间，那么也可以称为定义（Definition），因此定义是声明的一种

makefile

  • gcc hello.c -o hello
    • 编译 链接，生成hello可执行文件
    • ./hello
  • 多文件编译需要使用makefile，以上方法只适用于单文件编译

预处理

预定义都是以#开头

  • 宏定义
    • 结尾不要加分号，否则会一并进行替换；如果换行，需要加上\续航符
    • 无参数的宏可以作为常量，带参数的宏可以作为函数，在C语言中非常有用
    • 不带参数的宏定义有安全隐患，一般使用 const int 进行替换
  • 文件包含
    • 头文件或者其他源文件的引入
    • 一般大型项目包含许多源文件 头文件，通过引入相关文件，提高了模块的复用
  • 条件编译

模块化程序设计

  • 分块编译，分块开发

  • 多个源文件，多个头文件

  • .c .h

    • 源文件；头文件

  • 头文件设计原则
